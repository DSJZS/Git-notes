冲突产生的原因：
合并分支时，两个分支在同一个文件的相同行或者相邻行有两套完全不同的修改。Git无法替我们决定使用哪一个。必须人为决定新代码内容。





解决的方案：
冲突的文件中 Git 会标出冲突的部分

<<<<<<< 当前分支
=======
之间的内容是当前分支冲突部分的内容

=======
>>>>>>> 合并分支
之间的内容是合并分支冲突部分的内容

此时对冲突的部分进行删减，并且去掉 
<<<<<<< 当前分支
=======
>>>>>>> 合并分支
这三行标识

之后在Git Bash中
git add 冲突文件名
然后
git commit -m "日志信息"
(注意此时不要写文件名)

这样当前分支就成功合并了

*注意：合并分支依旧保持原样
